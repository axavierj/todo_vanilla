Description

A vanilla todo application. No framework other than on the back end is to be used. the limitation is native api only as far as something like react or angular. Libraries are ok as long as they are from a cdn. No node on the front end!

the main technologies used are:

Main features

  • log in and register

    log in and register to the application. an essential feature of any application. the log-out feature is also included.

  • crud operations

    a todo list is created. the list is saved in the database. the list is displayed in the application. the list is editable. the list items can be deleted.

  • search

    the list can be searched by title, description and person. the search is case insensitive.
